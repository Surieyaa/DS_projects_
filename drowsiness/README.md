# DriveGuard — Driver Drowsiness Detection

A CNN-based driver monitoring system that classifies driver safety state
(drowsy, yawning, distracted, drinking, closed eyes, dangerous / safe driving)
from video uploads or a live browser webcam feed, with color-coded alerts and
an audible alarm on sustained danger states.

## 2. Why one model, not two

Dataset 1 (DDD) only has `Drowsy` / `Non-Drowsy`. Dataset 2 has 7 finer-grained
classes. Rather than juggling two separate models in the Flask app,
`train.py` builds `dataset/combined/` — a single folder-per-class directory
(symlinks into both source datasets) — and trains **one** 9-class CNN:

`DangerousDriving, Distracted, Drinking, Drowsy, Non-Drowsy, Openeye, SafeDriving, Yawn, closed`

Each class is mapped to a risk bucket (`safe` / `warning` / `danger`) that
drives the UI color-coding and the alarm logic. That mapping lives in
`models/risk_map.json`, generated by `train.py`.

## 3. Train the model (Google Colab or local GPU)

Use the accompanying `driver_drowsiness_detection.ipynb` notebook, or run
locally:

```bash
pip install -r requirements.txt

# put the two Kaggle datasets under dataset/DDD and dataset/Thesis_dataset2
python train.py --ddd_dir dataset/DDD --thesis_dir dataset/Thesis_dataset2 --epochs 25
```

This produces:
- `models/drowsiness_cnn.h5`
- `models/labels.json`
- `models/risk_map.json`
- `models/training_history.png`

If you trained in Colab, download these three files (`.h5` + two `.json`)
and drop them into the local `models/` folder before running the Flask app.

## 4. Run the Flask app

```bash
pip install -r requirements.txt
python app.py
```

Then open `http://127.0.0.1:5000` in your browser.

- **Upload** (`/upload`): upload a video file, get back an annotated video,
  a class-distribution breakdown, and a list of alarm timestamps.
- **Live Monitor** (`/live`): click "Start camera" to grant webcam access.
  Frames are captured client-side and POSTed to `/predict_frame`; the
  response drives an on-screen bounding box, label, confidence ring, and a
  flashing alarm banner + beep once danger states persist for a few
  consecutive frames.

## 5. Notes on the live webcam design

Live detection runs the camera **in the browser** (`getUserMedia`) rather
than server-side `cv2.VideoCapture`, because the Flask server may be
deployed on a machine with no physical camera. Captured frames are sent to
`/predict_frame` roughly every 800&nbsp;ms as base64 JPEG; the server runs the
same `DrowsinessDetector` used for video upload and returns the label,
confidence and a face bounding box for the overlay.

## 6. Alert / alarm logic

- Video upload: an alarm event is logged once a `danger`-risk class persists
  for 5 consecutive analyzed frames (`DANGER_STREAK_THRESHOLD` in `app.py`).
- Live monitor: the on-screen alarm banner + beep trigger once 3 consecutive
  polled frames come back `danger` (`DANGER_STREAK_TO_ALARM` in `script.js`).

Both thresholds are simple constants you can tune.
